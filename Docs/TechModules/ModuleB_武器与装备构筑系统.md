# 模块 B：武器与装备构筑系统 (Weapon & Equipment Build System)

> **技术优先级：⭐⭐⭐⭐**
> 本模块依赖模块 A 的 `VisionLayer` 接口（武器注册/注销视界）。是 Roguelite 体验的核心承载体。

---

## 一、模块介绍

本模块管理玩家在"三选一 Roguelite"升级过程中获取并叠加强化的所有局内内容，包括：武器的槽位管理与等级升级、装备/被动的无上限叠加，以及通过词条 (Tags) 系统产生构筑化学反应 (Synergy) 的完整机制。

武器是"视界"的直接生成器，装备用于强化视界属性或战机基础属性。所有选项均围绕"视界 (Vision)"展开，玩家的终极目标是通过合理搭配，实现视界范围的最大化覆盖与锁定速度的极致叠加。

---

## 二、模块职责

- **武器槽位管理**：初始 4 槽（局外可解锁更多）；槽满时强制替换旧武器，旧武器产生的视界立即注销
- **武器等级系统**：重复选择已持有的武器 = 升级；视界的范围、锁定速度等属性按等级线性提升
- **视界生产接口**：每把武器持有 `VisionDefinition[]`，定义其产生的视界数组（形状/范围/持续时间/锁定速度/挂载位置偏移），调用模块 A 的 `VisionLayer` 完成注册
- **装备/被动系统**：无上限叠加；效果以词条 (Tags) 为键进行加法线性叠加
- **词条系统 (Tags)**：武器与装备通过共享词条产生 Synergy，如"激光"装备增强所有带"激光"词条武器的锁定速度
- **三选一随机池**：带权重的随机抽取（相同词条权重更高）；刷新机制（默认 1 次，局外可升至 3 次）
- **数据配置接口**：武器/装备均以 `ScriptableObject` 配置，策划零代码修改

## 三、对应需求原文

> 以下为《视界边缘》2.0 正式需求案原文，完整摘录，不做删减。

---

### 【需求原文】3.2 局内升级机制：三选一 (3-Choice System)

当战机升级时，游戏画面暂停，弹出"局内升级面板"。

- **选项池：** 系统会从当前解锁的"武器库"和"装备库"中随机抽取 3 个选项供玩家选择。

- **刷新机制：** 默认有 1 次刷新机会，永久升级里可以提升到 3 次。

- **稀有度设定：** 没有稀有度设定，但武器和装备会有词条设置，每个武器和装备都会有多个词条。随机抽取时，有相同词条的武器和装备抽到的可能性更大，方便玩家凑齐想玩的流派。

---

### 【需求原文】3.3 武器系统 (Weapons)

武器是**"视界"的直接生成器**。不同的武器会产生不同形状、持续时间和基础锁定速度的视界。

不管是武器还是被动装备，所有的升级与构筑都要围绕着**"视界 (Vision)"**展开。玩家的终极目标是通过合理的搭配，实现视界范围的最大化覆盖与锁定速度的极致叠加。

**武器槽位限制：** 战机初始及单局内最多只能同时装备 **4 个武器**（注：局外养成可解锁额外槽位，见后续模块）。

**获取与替换规则：**

- **槽位未满（< 4）：** 玩家在升级中选择新武器时，直接装备，在战机周围生成新的视界。

- **槽位已满（≥ 4）：** 玩家如果选择了一把新武器，**必须强制替换**身上已有的一个武器。被替换的武器其对应的视界立刻消失。

**武器的升级机制：**
- 武器具有等级，玩家在升级时选择已拥有的武器会增加武器等级。
- 武器的属性随等级线性提升。

**武器属性结构（示例）：**

策划在配置每一把新武器时，需明确以下字段：

```
武器 ID：          [如：WPN_001]
武器名称：         [填写武器名称]
视界生成类型：     [单视界生成 / 多视界生成]
生产视界的初始位置数组：位置坐标数组
生产视界数组：     视界数组
词条：             （如：激光，机械，智能，雷达）
```

**核心视界属性：**

```
形状 (Shape)：     [圆形 / 扇形 / 矩形 / 环形 / 异形(需特殊说明)]
范围/尺寸 (Range/Size)：[填写具体判定参数，如：半径 R=5，或 夹角 60 度+长度 10]
持续时间 (Duration)：   [常驻型 / 脉冲型(例如:存续 2 秒, CD 3 秒)]
锁定速度 (Lock-on Speed)：[X 点/秒。注: 速度越快，击破敌人隐蔽值越快]
所属词条 (Tags)：   [如：激光、机械、高频、广角等，用于后续产生化学反应]
视觉表现：         [UI 层面雷达/光环的颜色、扫图动画、以及发射出的导弹拖尾颜色等]
```

### 【需求原文】3.4 装备/被动系统 (Equipment / Passives)

装备用于**强化战机基础属性**或**改变已有视界属性**，是玩家产生化学反应（Synergy）的关键。

- **装备槽位限制：** 战机拥有**无上限**的装备槽位。

- **获取规则：** 升级时玩家可以重复获得已有的装备，装备名字后面会显示 +x 表示当前装备叠加数量。叠加的装备会导致装备效果中的数值按照数量简单累加。

**装备的分类与作用（举例）：**

- **视界强化类：** 例如"广角透镜"（所有扇形视界角度增加 30%）、"超频处理器"（所有视界的锁定速度增加 20%）、"持久供能"（所有周期性视界的持续时间延长 1 秒）。

- **基础属性类：** 例如"引力引擎"（经验拾取范围增加 50%）、"轻量化装甲"（战机移动与鼠标转向速度提升 15%）。

**被动装备设计模板：**

策划在配置每一件新装备时，需明确以下字段：

```
装备 ID：          [如：PAS_001]
装备名称：         [填写装备名称]
装备分类：         [基础属性强化类 / 视界机制改变类]
属性增益公式：
  目标属性：       [如：基础移速 / 经验吸附范围 / 特定形状视界的锁定速度]
  单次获取增益：   [如：+15% / +2 点]
  叠加逻辑：       [线性加法叠加：每多 1 件+X / 乘法递减叠加]
所属词条 (Tags)：   [如：引擎、装甲、超频、雷达等]
一句话描述 (Flavor Text)：[用于三选一界面展示的说明文案]
```

### 【需求原文】3.5 构筑策略与化学反应 (Build & Synergy)

结合模块二的规则（"区域重叠时锁定速度相加"），玩家在单局中的核心策略将是：

- **覆盖流：** 选择 4 把不同方向或极大范围的武器，确保屏幕上任何位置的敌人都会处于至少一个视界中，实现无死角防御。

- **重叠爆发流：** 选择多把具有指向性（如前方矩形、前方扇形）的武器。通过玩家缓慢的鼠标转向，将多个视界对准同一个高血量 Boss。在视界重叠区，锁定速度极快叠加，导致导弹如机关枪般连续发射。
